## üöÄ How It Works

### **Flow:**

1. **User logs in** ‚Üí JWT token stored in localStorage
2. **Socket connects** ‚Üí Frontend establishes WebSocket connection
3. **Socket authenticates** ‚Üí Backend verifies JWT token, joins user room
4. **Event triggers** ‚Üí Order placed, status updated, farmer registers, etc.
5. **Notification created** ‚Üí Backend saves to MongoDB
6. **Socket emits** ‚Üí Sends notification to specific user's room
7. **Frontend receives** ‚Üí NotificationContext updates state
8. **Toast displays** ‚Üí User sees popup notification
9. **Bell updates** ‚Üí Unread count badge increments
10. **User clicks** ‚Üí Opens dropdown, sees all notifications
11. **Mark as read** ‚Üí Updates database and UI

## üì° Socket.io Configuration

### Backend (`server.js`):
```javascript
const io = new Server(server, {
  cors: {
    origin: 'http://localhost:5173',
    methods: ['GET', 'POST'],
    credentials: true
  }
})

// User joins their own room
socket.join(`user_${userId}`)

// Emit to specific user
io.to(`user_${userId}`).emit('notification', data)
```

### Frontend (`NotificationContext.jsx`):
```javascript
const socket = io('http://localhost:5000', {
  autoConnect: false
})

socket.emit('authenticate', token)
socket.on('notification', handleNewNotification)
```
